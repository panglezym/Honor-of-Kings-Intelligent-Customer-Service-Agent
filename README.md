## ✨ 项目简介

这是一份面向生产环境的 **多 Agent 智能客服系统** 设计稿，以王者荣耀（Honor of Kings）为业务场景，完整覆盖从 NLU 意图识别 → 任务派发 → 专域执行 → 规则核验 → 语言适配的全链路设计。

与普通的 PPT 设计稿不同，本项目把 **可点击的交互演示** 直接嵌入文档，点击场景按钮即可看到 Agent 的思考过程、工具调用、知识图谱多跳检索、规则核验决策——所见即所得。

适合：
- AI 产品经理想学习真实客服 Agent 的完整系统设计
- 后端/算法同学了解多 Agent 编排与降级链路的工程落地
- 面试展示 / 技术分享 / 系统设计案例

## 🏛 核心架构

```
┌──────────────────────────────────────────────────────┐
│  L1 · 编排层                                          │
│  Orchestrator Agent · Context Manager                │
└────────────┬─────────────────────────────────────────┘
             │ A2A Protocol · 任务并行派发
┌────────────┴─────────────────────────────────────────┐
│  L2 · 专域执行层（5 个 Agent · 并行执行）              │
│  账号安全 · 支付退款 · 对局补偿 · 游戏内容 · 规则政策    │
└────────────┬─────────────────────────────────────────┘
             │ 按区服语言注入差异化规则
┌────────────┴─────────────────────────────────────────┐
│  L3 · 语言适配层                                      │
│  中文 · 繁中 · 东南亚 · 英语                           │
└──────────────────────────────────────────────────────┘
```

- **14 个知识图谱节点** · 39 条关系边 · 支持多跳推理
- **Policy 作为 SSOT**（Single Source of Truth），规则变更时只改适配器
- **三层韧性保护**：API 熔断 · Redis 热缓存兜底 · 置信度 + 情绪双阈值升人工

## 💡 设计亮点

- **多 Agent 并行 vs 单 Agent 串行的取舍**：复合问题时延从 2.3s 压到 1.2s，但牺牲了 Orchestrator 的调度复杂度
- **降级链路的完整性**：失败时如何优雅升级，而不是"强行回答"——涉及账号安全类问题的医疗伦理级底线
- **多语言的真正价值**：同一个问题，国服玩家不能退、泰国玩家可以退——关键在规则继承而非翻译
- **情绪前置处理**：愤怒分值 0.82 时主动降低升人工阈值至 0.75

每个设计决策都在演示中附带 **权衡取舍说明**。
